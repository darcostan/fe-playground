📚 Lifecycle Hook Reference

ngOnInit
Called once after the component is initialized. Use for setup logic, data fetching.
ngOnChanges
Called when input properties change. Receives SimpleChanges with previous & current values.
ngAfterViewInit
Called after component's view and child views are initialized. Access ViewChild here.
ngAfterContentInit
Called after content projected into the component is initialized (ng-content).
ngOnDestroy
Called just before the component is destroyed. Clean up subscriptions, timers here.
ngAfterViewChecked
Called after every check of the component's view. Use sparingly — runs frequently.

💻 Implementation

export class MyComponent implements OnInit, OnDestroy, AfterViewInit {
  @ViewChild('myRef') myRef!: ElementRef;

  ngOnInit() {
    // ✅ Setup, fetch data, initialize state
  }

  ngAfterViewInit() {
    // ✅ Access ViewChild references
    console.log(this.myRef.nativeElement);
  }

  ngOnDestroy() {
    // ✅ Clean up: unsubscribe, clear timers
  }
}
